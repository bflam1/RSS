Mend SCA: Version 25.7.1 (27-July-2025)

2025-07-27T00:00:00+00:00

New Features and Updates

  • Introducing enhanced security visibility: View security violations directly within your libraries. New columns display violation counts, sortable and filterable, in both the main table and side panels, providing a comprehensive risk assessment for efficient prioritization and action.

  • Introducing new statuses in the Mend AppSec Platform for better tracking and alignment with remediation processes, maintaining backward compatibility with the Legacy SCA application. Added statuses include "In Review" and "Issue Created" for direct libraries. Existing status behaviors remain unchanged, ensuring seamless integration and workflow continuity across platforms.

Mend SAST: Version 25.6.2 (14-July-2025)

2025-07-14T00:00:00+00:00

New Features and Updates

  • The workflow engine now allows users to use compliance standard violations directly as conditions, without needing to configure CWEs manually.

  • Introduced handling of historical findings older than 90 days. This ensures that previously suppressed findings will not reappear as new issues.

  • Introduced a detailed progress indicator for the code analysis in the CLI, giving users better visibility into scan progress.

  • Findings reports now include Violations, "Endpoint Access", and "Exploitable" data columns, enabling AppSec managers to efficiently assess and prioritize risks.

Resolved Issues

  • Fixed an issue where multi-line statements with a lambda function showed the wrong line number, pointing to the start of the entire statement instead of the lambda. Now, the line number shows exactly where the lambda is, so its easier to find and fix.

  • Fixed an issue where temporary variable names in the trace caused findings to mismatch with previous scans. This sometimes led to previously suppressed findings being incorrectly reported again.

Mend Developer Platform: Version 25.7.1 (27-July-2025)

2025-07-27T00:00:00+00:00

New Features and Updates

  • (Open Beta) The Developer Platform now supports custom configuration, partitioning, and scanning for monorepos. Applicable for both SCA and SAST.

  • (SCA) Poetry 2.x project scans are now supported.
    This enhancement allows users to analyze dependencies defined and locked with Poetry 2.x using the same workflow, at no additional configuration overhead.

Resolved Issues

  • Fixed a bug where non-admin users could click on the +Add button in the custom work items section of the Developer Platform settings. The button is now disabled for non-admins.
