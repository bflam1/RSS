Mend Unified Agent: Version 25.7.2 (11-August-2025)

2025-08-11T00:00:00+00:00

Resolved Issues

  • Improved the user experience by updating the screening process so only files that are used for dependency resolution will be logged, preventing misleading messages.

Mend Developer Platform: Version 25.8.1 (24-August-2025)

2025-08-24T00:00:00+00:00

New Features and Updates

  • Pull requests to release branches (configured as such in the Developer Platform settings) will now be scanned by default.

  • The Bitbucket integration now enables users to configure multiple workspace tokens which will be used in rotation, to avoid hitting Bitbuckets API rate limits.

  • New columns for SCA, SAST, and Renovate have been added to the repositories table in the Developer Platform UI, allowing users to sort repositories by engine status (Enabled/Disabled, based on organizational entitlements).

  • The Developer Platform now supports custom configuration, partitioning, and scanning for monorepos, in both SCA and SAST.
    This feature has been in open beta since 25.7.1 and is now generally available.

Resolved Issues

  • Improved the handling of scenarios where the user has multiple memberships for an organization in Bitbucket.

  • Improved the process for manually triggering scans. The available branches for selection will now be based on the repository settings if those override the global settings.

  • Fixed a bug where the Load more button in the Developer Platform homepage failed to display all Azure DevOps projects.

Mend for GitHub Enterprise: Version 25.8.1.1 (24-August-2025)

2025-08-24T00:00:00+00:00

Unified Agent 25.4.3-179 | Renovate 41.71.1 | Remediate 25.8.1 | Pre-Scan Builder (PSB) 25.8.1

New Features and Updates

  • The First Fix remediation strategy is now available for Python in self-hosted repository integrations. Enable by setting the environment variable MEND_PYTHON_FIRSTFIX_ENABLED to true (default is false).

  • Introducing a new capability for executing custom pre-scan scripts, enabling users to prepare the environment or perform setup tasks before scanning.

Resolved Issues

  • Fixed an issue where system memory was unnecessarily seized by the Gradle daemon after executing a Gradle script during a scan. Memory handling has been improved for increased efficiency.

  • (SAST) Fixed a bug in the proxy configuration of the repository integration which erroneously
    led to SAST scans not being triggered and a message about incorrect SAST credentials.
