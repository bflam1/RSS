Mend AppSec Platform: Version 25.10.3 (16-November-2025)

2025-11-16T00:00:00+00:00

New Features and Updates

  • Introducing a Risk Reduction Dashboard focused on security findings. This tool aids security teams by highlighting relevant issues, enabling swift action, and showcasing potential risk reduction opportunities. Features include a risk funnel and application/project tables with actionable insights, delivering a streamlined, confident user experience.

  • Added the Legal Manager role to the platform (License and Copyright Assigners in Legacy SCA), enabling organizations to assign license and copyright management permissions directly through the UI. This enhancement streamlines user management and supports a more granular permission model, aligning with least-privilege access and role-based administration.

    • License and Copyright Assigners assignments in Legacy SCA will be reflected as Legal Manager role assignments in the Mend AI-Native AppSec Platform and vice versa.

  • (SCA) Automation Workflows now apply in every CLI dependencies scan, regardless of whether the --update parameter is specified in the scan command or not.

    • This update has been in limited availability since Oct 27th and is now generally available.

Mend SAST: Version 25.10.3 (17-November-2025)

2025-11-17T00:00:00+00:00

New Features and Updates

  • Mend for Visual Studio Code is now available!
    The extension provides developers with immediate information about security findings and remediation suggestions directly in the IDE, before the code is committed into an SCM.

    As a result, securing applications becomes a lot simpler, as developers can perform crucial actions to triage security findings without navigating away from the IDE.

  • Mend SAST findings can now be uploaded into Wiz. The integration allows security teams to correlate static vulnerabilities with runtime context in cloud assets. This enhances prioritization by identifying exploitable issues based on real-world exposure.

  • Improvements have been made to enhance detection accuracy of vulnerabilities related to Rust projects, strengthening overall security coverage. In detail, the following CWEs were improved:

    • CWE-614: Sensitive Cookie Without Secure

    • CWE-943: NoSQL Injection

    • CWE-1004: Cookie Without 'HttpOnly' Flag

  • Addressed macro expansion handling in C/C++ (Gen 2), ensuring method names and data flow traces accurately reflect the source code. Also, improved the representation of temporary variables for clearer, more reliable analysis results.

  • SAST secret scanning now includes .xcconfig files by default, detecting hardcoded API keys and sensitive data in Xcode project files.

  • The SAST scanner now automatically excludes Gradle wrapper distribution files from scans, reducing false positives. This change minimizes manual configuration and enhances user experience by eliminating irrelevant findings not part of the source code.

Mend Container: Version 25.10.3 (16-November-2025)

2025-11-16T00:00:00+00:00

New Features and Updates

  • General CLI container scan performance improvements.

Resolved Issues

  • Resolved a a log output issue during scans: By removing specific code lines, log messages no longer print to STDOUT/STDERR when MEND_LOG_LEVEL is set. Console logging remains at ERROR level, except in development mode.

  • Resolved an issue in Desktop Printing Software for Linux where HTML reports for Container Image Attribution displayed an incorrect number of libraries, exceeding the project's parameters when generating reports in the customer's environment.

  • Fixed an issue where violations generated by workflows were not automatically resolved when their associated findings were suppressed. Now, violations are promptly resolved across all scan types when findings are suppressed, ensuring consistent and immediate updates.

  • Fixed an issue where users with the Security Analyst role were unable to suppress container findings via API v3 without admin permissions. Suppression can now be performed by users with appropriate non-admin roles.
