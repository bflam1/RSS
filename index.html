Mend.io Unified Release Notes

Mend AppSec Platform: Version 26.2.1 (22-February-2026)

2026-02-22T00:00:00+00:00

New Features and Updates

  • (SCA) Source File Matching in Mend Platform introduces automatic and manual grouping of source files into logical libraries, improving vulnerability detection, license attribution, and compliance reporting. This update adds a new Source File column, a dedicated Source Files page, and manual override options, streamlining workflows for Security and Legal teams while ensuring more accurate and actionable insights.

  • UI improvements:

    • Standardized the placement and design of refresh and update indicators across all main pages for a more consistent and intuitive user experience.

    • Added a rows-per-page selector to paginated tables, allowing users to choose between 50 (default) and 100 rows per page directly from the pagination footer.

      This improves usability by reducing page navigation and making it easier to review larger datasets.

Resolved Issues

  • Fixed an issue where tag values were not displayed in the scan summary, even though they appeared correctly in project scans. Tag values now consistently show in both views.

  • Fixed an issue where users switching from the Legacy UI to the Mend AppSec Platform were redirected to the login screen instead of being automatically authenticated, under certain conditions.

  • Fixed an issue where users could not search for all Jira projects in the mappings input fields of the Legacy UI.

  • Fixed an issue where importing platform OpenAPI schema definitions into API management tools resulted in syntax errors.

Mend SCA: Version 26.2.1 (22-February-2026)

2026-02-22T00:00:00+00:00

New Features and Updates

  • Source File Matching in Mend Platform introduces automatic and manual grouping of source files into logical libraries, improving vulnerability detection, license attribution, and compliance reporting. This update adds a new Source File column, a dedicated Source Files page, and manual override options, streamlining workflows for Security and Legal teams while ensuring more accurate and actionable insights.

  • NVIDIA Proprietary Software License is now automatically detected and classified as proprietary.

Resolved Issues

  • Fixed an issue where certain libraries were incorrectly flagged with CVE-2021-41248.

  • Fixed an issue where the Attribution Report did not display license texts for some licenses.

  • Fixed an issue where the SPDX SBOM report would erroneously display NOASSERTION or NONE for the licenseConcluded and licenseDeclared fields.

  • Fixed an issue where certain Intel MKL libraries were not properly assigned to the "Intel Simplified Software License". The license has now been added and these libraries are correctly associated with it.

Mend AI: Version 25.12.1 (05-January-2026)

2026-01-05T00:00:00+00:00

New Features and Updates

  • (Premium / Core) Introducing System Prompt Risk (open beta), a new detection and remediation offering by Mend AI, for mitigating risks posed by system prompts used in conversational AI interfaces.

    • A new System Prompt Risk table inventories system prompts and provides quick export/sharing and deep-link side panel for prompt context.

    • System Prompt Risks are integrated into the existing AI Security Risk Factors across Projects and Applications. The classification appears as a Conversational Interface chip, is filterable, and automatically participates in dashboards and workflows.

    • Remediation in the form of a hardened system prompt is available in the System Prompt side panel, providing clear, copy-ready remediation guidance for AppSec engineers and developers.

    • The AI Security Dashboard has been enriched with system prompt risk data.

    • System prompt risk data is also available via API.

Mend Unified Agent: Version 26.2.1 (23-February-2026)

2026-02-23T00:00:00+00:00

Resolved Issues

  • Fixed an issue where scanning large Docker container images in OCI tar format could fail silently due to unnecessary duplication of the .tar file, which exhausted disk space and resulted in incomplete scans with no reported dependencies.

  • Fixed an issue where Docker image scans failed when using the new Docker CLI output format. The Unified Agent now correctly parses image IDs, ensuring successful vulnerability scans for Docker images regardless of Docker CLI changes.

Mend Developer Platform: Version 26.2.1 (22-February-2026)

2026-02-22T00:00:00+00:00

New Features and Updates

  • (AZDO) The Developer Platform will now alert Azure DevOps integration users with a banner when their authentication to Azure is no longer working, and provide a button to reconnect.

Resolved Issues

  • Fixed an issue where Gradle dependencies failed to resolve for React Native Android projects, ensuring accurate detection and resolution of dependencies by properly handling project structure and Gradle wrapper logic.

  • Fixed an issue where API calls with heavy processing would fail due to insufficient timeout settings. The HTTP request timeout has been increased to better support longer-running operations.

  • Fixed an issue where no IN_PROGRESS check run was set when a delayed Reachability scan was added, ensuring users now receive accurate scan status updates during delayed scans.

Mend for GitHub.com: Version 26.1.2 (08-February-2026)

2026-02-08T00:00:00+00:00

Resolved Issues

  • Fixed an issue where Gradle dependencies failed to resolve for React Native Android projects, ensuring accurate detection and resolution of dependencies by properly handling project structure and Gradle wrapper logic.

  • Fixed an issue where the scanner could get stuck in a loop after a scan timed out, repeatedly logging timeout errors instead of properly cancelling the task. The process now ensures scans are cancelled as expected, improving reliability for long-running scans.

Mend for GitHub Enterprise: Version 26.1.1 (25-January-2026)

2026-01-25T00:00:00+00:00

Unified Agent 25.11.1-223 | Renovate 42.74.5 | Remediate 26.1.1 | Pre-Scan Builder (PSB) 25.8.1

New Features and Updates

  • The repo cloning default utility has been changed from Jgit to Git Shell. This is controllable via the WS_GIT_CONNECTOR scanner environment variable.

Resolved Issues

  • Fixed an issue where marking a license alert as Ignored in the Mend Platform would not prevent the integration from listing it as a policy violation in the License Check run and auto-creating a GitHub issue for it.

  • Fixed a bug where PR scans were skipped when the PR originated from a cross-org fork.

  • Upgraded the "cryptography" library in the scanner from version 42.0.2 to 46.0.3, to fix CVE-2024-26130.

  • Fixed an authentication issue which led to project installation errors in npm scans if non-default ports were configured to communicate with the package registry. The scanner now correctly preserves the port in the .npmrc file, ensuring proper credential matching and preventing installation errors for projects using custom registry setups.
