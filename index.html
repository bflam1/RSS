Mend AppSec Platform: Version 25.12.1 (28-December-2025)

2025-12-28T00:00:00+00:00

New Features and Updates

  • Introducing a natural language search bar powered by AI, enabling users to filter applications and projects with intuitive prompts. This feature streamlines finding critical information, offers smart filter suggestions, and enhances the overall user experience with quick, relevant results and improved filter management.

  • Enhanced SAML integration with new configuration options, improved user provisioning controls, and a modernized UI for a more flexible and secure authentication experience. Updates include, but are not limited to, the following enhancements:

    • Disabling local logins

    • Toggling just-in-time user creation

    • Uploading or editing metadata in multiple formats

    • Notifications and visual indicators when the SAML certificate is close to expiration

    • A two-way SSO enforcement across the Mend AppSec Platform and Legacy SCA application.

Mend SAST: Version 25.11.2 (15-December-2025)

2025-12-15T00:00:00+00:00

New Features and Updates

  • Improved accuracy of Java, JavaScript/TypeScript and Python analysis, especially for the following CWEs:

    • CWE-78: Command Injection

    • CWE-79: Cross-site Scripting

    • CWE-89: SQL Injection

  • Added support for two compliance standards, PCI DSS 4.0 and MISRA, now visible in the code findings table and code reports.

  • Dead code analysis for Java now excludes generated files and skips scanning those exceeding the maximum file size limit, improving performance and focusing results on relevant code only.

  • Enhanced support for Rust macros, ensuring they are properly parsed to detect tainted data.

  • When reviewing suppression requests, users can now filter by predefined reasonsfalse positive or acceptable riskto make the review process more efficient.

  • Newly created organizations will now have C++ gen 2 engine enabled by default, ensuring all users benefit from the latest version automatically.

  • Scans are now automatically stopped if the CLI cannot retrieve required engine configuration, preventing inconsistent results when some components are unreachable.

  • Added the ability to filter, sort, and group findings by project in the applications view, making it easier to manage and analyze security findings.

Resolved Issues

  • Fixed an issue where the PHP engine could become unresponsive on Windows due to improper timeout handling. The timeout mechanism has been adjusted to restore previous logic on Windows, ensuring stable scanning and preventing unexpected engine hangs.

  • Fixed an issue where importing results from an external SARIF file would fail if the tool name in the imported file contained spaces.

  • Fixed an issue that caused reports to show negative numbers of detected findings.

Mend CLI: Version 25.11.2.2 (22-December-2025)

2025-12-22T00:00:00+00:00

New Features and Updates

  • (SCA) Hotfix:

    • The SCA CLI now supports tags with increased key and value lengths up to 255 characters, removes the previous 20-tag limit, and allows a broader range of characters.

    • Added a new CLI parameter, --persist-logs, that allows users to persist all log levels, including INFO, to a specified directory after every SCA scan. This enhancement ensures comprehensive log availability for auditing, compliance, and integration with external log management systems, regardless of scan outcome.

Mend Unified Agent: Version 25.11.2 (15-December-2025)

2025-12-15T00:00:00+00:00

Resolved Issues

  • Fixed an issue where invalid system paths in certain PNPM and Yarn projects caused Reachability analysis to fail.

  • Fixed an issue where Go projects with build errors caused all dependencies to appear as direct in the UI, resulting in the loss of transitive dependency hierarchy and breaking vulnerability path analysis. Dependency relationships are now accurately reported even when fallback mechanisms are triggered.

  • Fixed an issue where Erlang dependencies could not be resolved during project scans, resulting in incomplete dependency trees.

  • Fixed an issue where Elixir dependencies could not be resolved during project scans, under certain conditions.
