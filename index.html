Mend.io Unified Release Notes

Mend AppSec Platform: Version 26.4.1 (19-April-2026)

2026-04-19T00:00:00+00:00

New Features and Updates

  • Introducing the Users & Groups report, allowing admins and security teams to generate comprehensive reports of all users, their roles, and group memberships within an organization. Reports can be generated from the UI or via API, supporting multiple formats for streamlined auditing, compliance, and user management.

  • Added support for recognizing CPE (Common Platform Enumeration) identifiers during SBOM imports in both SPDX and CycloneDX formats. When a PURL is missing, the importer now uses CPEs to identify components, ensuring broader compatibility and successful imports even when only CPEs are provided.

  • Enabled Application Admins to remap source files within their assigned applications, improving operational scalability and reducing reliance on organization admins. Application Admins are now restricted to application-level remapping, with UI and API updates ensuring clear permission boundaries and consistent role-based access control.

  • UI Improvements:

    • Revamped the organizational Findings table with a streamlined column structure, improved filtering and sorting options, and enhanced search functionality. The update introduces clearer terminology, modern UX patterns, and a more intuitive interface to simplify vulnerability analysis and boost usability.

    • Enhanced the Report Page UI for improved visual consistency and usability. Updates include refreshed scan engine icons, optimized column layouts, ellipsis overflow for text cells, and unified status chip designs to align with current UX/UI standards. The "Generated By" cell is now hidden by default for a cleaner look.

Mend SCA: Version 26.3.2 (12-April-2026)

2026-04-12T00:00:00+00:00

New Features and Updates

  • Introducing new Zero-Day response workflows in the Mend AppSec Platform, offering immediate risk visibility, audit-level transparency, and user-controlled enforcement for critical vulnerabilities. Users benefit from prominent alerts, a dedicated Zero-Day catalog, configurable workflows, and permanent violation tracking—streamlining investigation and remediation without added operational burden.

  • Improved license accuracy of Maven libraries. Updates include clearer license version specificity, removal of redundant entries, and detection of previously unknown licenses.

  • The Due Diligence report can now be generated in Excel format.

Resolved Issues

  • Fixed an issue where certain "software.amazon.awssdk" components displayed duplicated or incorrect copyright statements. All affected components now show accurate and consistent copyright information.

Mend SAST: Version 26.4.1 (20-April-2026)

2026-04-20T00:00:00+00:00

New Features and Updates

  • (Closed Beta) Introducing the Gen 2 Go detection engine, with improved CWE and framework coverage for accurate, scalable scans.

  • Automated remediation suggestions for Code findings are now also supported for Rust. The following CWEs are covered:

    • CWE-22 - Path Traversal

    • CWE-78 - Command Injection

    • CWE-89 - SQL Injection

    • CWE-90 - LDAP Injection

    • CWE-643 - XPath Injection

    This feature will be rolled out gradually during the next 14 days.

  • Added support for additional low-probability security protocols in Rust, expanding coverage for several CWEs and enhancing detection capabilities.

  • Several accuracy improvements for C#, mainly for the following CWEs:

    • CWE-89: SQL Injection

    • CWE-502: Deserialization of Untrusted Data

    • CWE-643: XPath Injection

  • Improved the accuracy of CWE-79: Cross-site Scripting for Angular.

  • Incremental scanning is now available for secrets detection. When running a scan, only files that have changed since the last scan are analyzed, reducing scan times.

  • CSV exports and generated reports now include a Deep Link column with a direct URL to each finding in the Mend AppSec Platform UI, making it faster to navigate from a report to the relevant finding for triage or remediation.

Resolved Issues

  • Fixed an issue where temporary folders were not cleaned up properly after a CLI scan, which could lead to scan failures in certain cases.

Mend AI: Version 25.12.1 (05-January-2026)

2026-01-05T00:00:00+00:00

New Features and Updates

  • (Premium / Core) Introducing System Prompt Risk (open beta), a new detection and remediation offering by Mend AI, for mitigating risks posed by system prompts used in conversational AI interfaces.

    • A new System Prompt Risk table inventories system prompts and provides quick export/sharing and deep-link side panel for prompt context.

    • System Prompt Risks are integrated into the existing AI Security Risk Factors across Projects and Applications. The classification appears as a Conversational Interface chip, is filterable, and automatically participates in dashboards and workflows.

    • Remediation in the form of a hardened system prompt is available in the System Prompt side panel, providing clear, copy-ready remediation guidance for AppSec engineers and developers.

    • The AI Security Dashboard has been enriched with system prompt risk data.

    • System prompt risk data is also available via API.

Mend Unified Agent: Version 26.3.2 (14-April-2026)

2026-04-14T00:00:00+00:00

Resolved Issues

  • Fixed an issue where the Unified Agent failed to detect npm dependencies in Yarn Berry (v4) projects using workspaces. The scanner now correctly identifies and processes workspace package.json files and root yarn.lock files, ensuring all dependencies are accurately reported in scan results.

  • Fixed an issue where configuring python.pipPath to an absolute path caused setup.py dependency resolution to detect only one dependency instead of the full dependency tree. All dependencies are now correctly identified during scans when using an absolute pip path.

Mend Developer Platform: Version 26.4.1 (19-April-2026)

2026-04-19T00:00:00+00:00

Resolved Issues

  • Fixed an issue where deleting an account or refresh token could fail partway through for users with many installations, leaving some projects untouched. The removal process is now asynchronous and processes each installation independently, ensuring all installations are handled and providing clear per-installation results.

  • (SAST) Fixed an issue where suppression requests for findings that no longer exist in the latest project scan were rejected with a generic error message. The system now provides clear feedback, informing users when a suppression command fails because the finding is not present in the current state of the project.
