Mend.io Unified Release Notes

Mend AppSec Platform: Version 26.1.2 (08-February-2026)

2026-02-08T00:00:00+00:00

Resolved Issues

  • Fixed an issue where Jira tickets created by the Mend for Jira Cloud integration did not receive comments when related Mend alerts were resolved, ensuring resolution status is now properly communicated within Jira.

  • Fixed an issue where the path to the dependency file was missing in the UI for some dependencies. File locations are now displayed consistently.

  • (SCA) Fixed an issue where importing a valid SBOM could fail without displaying an error message. Users will now receive clear feedback if an import fails.

New Features and Updates

  • UI improvements:

    • The top navigation has been redesigned to be fully responsive, ensuring all menu items remain accessible without horizontal scrolling. Buttons that do not fit are grouped under a more dropdown, and profile and admin options are now consolidated into a single menu. Updated styling and hover effects provide a modern, consistent look across the platform.

    • Updated the sidebar across the platform with a modern design, improved selected tab visibility, and consistent collapse/expand behavior. The new sidebar enhances usability and accessibility, making navigation clearer and more user-friendly for all users.

Mend SCA: Version 26.1.2 (08-February-2026)

2026-02-08T00:00:00+00:00

New Features and Updates

  • Improved detection and vulnerability analysis of R Language, Rust and PHP (composer) libraries.

Resolved Issues

  • Fixed an issue where certain libraries were incorrectly flagged with false positive GPL license violations due to misidentification of license exceptions. Improved license detection now ensures accurate reporting and prevents unnecessary compliance alerts.

  • Fixed an issue where importing a valid SBOM could fail without displaying an error message. Users will now receive clear feedback if an import fails.

Mend SAST: Version 26.1.2 (09-February-2026)

2026-02-09T00:00:00+00:00

New Features and Updates

  • Added the ability to generate SAST reports that include resolved (historical) findings for a selected time frame. Users can now export these reports in CSV, XML, and JSON formats via both API and Platform UI, enabling better tracking of risk trends and remediation effectiveness for security and compliance needs.

  • Increased the accuracy of CWE-79: Cross-site Scripting detection for JavaScript.

  • Multiple minor improvements to increase the accuracy of Rust detection.

  • Improved secret scanning to better detect hardcoded database connection strings.

  • Changes in configuration files now trigger secret scanning even when no actual code is present in the repository.

  • Repository integration users can now request to disable Secure Code Warrior training materials by contacting Mend Support.

Mend Container: Version 26.1.2 (08-February-2026)

2026-02-08T00:00:00+00:00

New Features and Updates

  • Kubernetes system namespaces can now be included in the scan using a dedicated toggle in the configuration of your Kubernetes integration in the Mend AppSec Platform UI.

  • Added support for updating credentials for existing Registry and Kubernetes integrations.

  • Optimized the scanner to clean up image files during the scan process, significantly reducing storage usage and improving resource efficiency.

Mend AI: Version 25.12.1 (05-January-2026)

2026-01-05T00:00:00+00:00

New Features and Updates

  • (Premium / Core) Introducing System Prompt Risk (open beta), a new detection and remediation offering by Mend AI, for mitigating risks posed by system prompts used in conversational AI interfaces.

    • A new System Prompt Risk table inventories system prompts and provides quick export/sharing and deep-link side panel for prompt context.

    • System Prompt Risks are integrated into the existing AI Security Risk Factors across Projects and Applications. The classification appears as a Conversational Interface chip, is filterable, and automatically participates in dashboards and workflows.

    • Remediation in the form of a hardened system prompt is available in the System Prompt side panel, providing clear, copy-ready remediation guidance for AppSec engineers and developers.

    • The AI Security Dashboard has been enriched with system prompt risk data.

    • System prompt risk data is also available via API.

Mend CLI: Version 26.1.2.3 (09-February-2026)

2026-02-09T00:00:00+00:00

New Features and Updates

  • (SCA) Added Reachability support to the CLI in self-contained mode, allowing users to run reachability checks without external dependencies. All required artifacts are now bundled and extracted for all supported operating systems, improving usability for air-gapped and enterprise environments.

  • (SCA) Added a new CLI log parameter that allows users to print logs directly to the console, making it easier to monitor activity and understand process flow in real time.

Resolved Issues

  • (SCA) Fixed an issue where post-scan cleanup and statistics were not executed if the scan process ended with an error, ensuring proper handling and reporting even when scans fail.

  • (SCA) Fixed an issue where scans relying on certain package managers failed, under certain conditions.

Mend Unified Agent: Version 26.1.2 (09-February-2026)

2026-02-09T00:00:00+00:00

Resolved Issues

  • Fixed an issue where Unified Agent did not apply configured proxy settings when downloading missing Maven dependencies from Maven Central, causing connection timeouts and extended scan times in environments requiring proxy usage. Now, all Maven Central downloads correctly respect proxy configurations.

  • Fixed an issue where the Unified Agent failed to download missing Maven dependencies due to use of an incorrect Maven Central URL, resulting in 403 errors.

  • Fixed an issue where the Unified Agent failed to properly parse pnpm lockfiles containing tarball URLs, resulting in missing dependencies due to a YAML parsing error. The parser now correctly handles these lockfiles and logs detailed error messages when parsing fails.

  • Fixed an issue where enabling docker.excludeBaseImage failed to exclude base image layers for Docker images without CMD instructions in their history, resulting in all layers being scanned and all base image packages reported without warning.

Mend for GitHub.com: Version 26.1.2 (08-February-2026)

2026-02-08T00:00:00+00:00

Resolved Issues

  • Fixed an issue where Gradle dependencies failed to resolve for React Native Android projects, ensuring accurate detection and resolution of dependencies by properly handling project structure and Gradle wrapper logic.

  • Fixed an issue where the scanner could get stuck in a loop after a scan timed out, repeatedly logging timeout errors instead of properly cancelling the task. The process now ensures scans are cancelled as expected, improving reliability for long-running scans.

  • Fixed a bug which led to scans running for more than 6 hours and timing out.

Mend for GitHub Enterprise: Version 26.1.1 (25-January-2026)

2026-01-25T00:00:00+00:00

Unified Agent 25.11.1-223 | Renovate 42.74.5 | Remediate 26.1.1 | Pre-Scan Builder (PSB) 25.8.1

Resolved Issues

  • Fixed an issue where marking a license alert as Ignored in the Mend Platform would not prevent the integration from listing it as a policy violation in the License Check run and auto-creating a GitHub issue for it.

  • Fixed a bug where PR scans were skipped when the PR originated from a cross-org fork.

The scanner now correctly preserves the port in the .npmrc file, ensuring proper credential matching and preventing installation errors for projects using custom registry setups.
