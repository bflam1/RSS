Mend.io Unified Release Notes

Mend AppSec Platform: Version 25.10.1 (20-October-2025)

2025-10-20T00:00:00+00:00

New Features and Updates

  • The Mend Security Assistant MCP is now available for both SAST and SCA!
    The assistant is an innovative tool integrated into AI-native IDEs, enabling developers to detect and fix vulnerable libraries or pieces of code in real-time.
    This offering allows developers to maintain secure coding practices seamlessly within their existing workflow, securing dependencies and code without interruption and building greater trust in generated outputs. Mend Security Assistant MCP is available for:

    • Amazon Q

    • Claude Code

    • Cursor IDE

    • VS Code Copilot

    • Windsurf IDE

  • Added a new Scan Metadata operational triggering event, enabling users to leverage tags as workflow conditions. You can now define workflows using Scan Tag (Key) or Scan Tag (Key:Value) with Matches and Doesn't Match operatorsideal for assigning scope, labeling branches, or aligning policies by tag.

  • Introducing Persona-Based Violation Filtering: Security and Legal teams can now view violations separately, ensuring each persona sees only relevant risks. Toggle between views or combine them for a unified perspective.

  • Users can now suppress violating findings directly from the main table and side-panel - at the Origin level (Root Library for SCA, Package for Containers), Library level, and Finding level - for improved remediation tracking across SCA, Containers, and Code.

  • Added an Automatic User Creation toggle in the SAML settings. Admins can now choose whether new users are created automatically on first SSO login or must be added manually.

Mend SCA: Version 25.10.2 (02-November-2025)

2025-11-02T00:00:00+00:00

Resolved Issues

  • Resolved an issue where the originator field in exported SBOM reports was incorrectly set to NOASSERTION even when valid author information was available for CDNJS packages.

  • Fixed an issue where users attempting to mark a library as Open Source in the Mend Platform, when that library had been previously defined as In-House in the Legacy SCA Platform, encountered only a generic Error message.
    With this fix, a clear pop-up message now appears, explaining that the action is blocked due to existing In-House rules in the Legacy SCA Platform. The prompt also provides a direct link and guidance to update the In-House rules before reclassifying the library as Open Source.

Mend SAST: Version 25.10.1 (20-October-2025)

2025-10-20T00:00:00+00:00

New Features and Updates

  • Introducing the Gen 2 C/C++ detection engine, with improved CWE and framework coverage for accurate, scalable scans.

  • (Limited Availability) Introducing a new approval process for security finding suppressions in code repositories. Developers can propose suppressions, while security managers can approve or reject them, ensuring control without development delays. This bridges developer and security workflows, enhancing efficiency.

  • Improved the performance of PHP analysis in some scenarios.

  • When creating the first pull request that includes findings in a repository with no prior scans on the base branch, the security check will now fail, and those findings will not be considered backlog findings. This change enables a smoother onboarding process by allowing new repositories to be initialized through the first pull request instead of requiring a direct commit to the main branch.

  • The onboarding configuration template for SCMs using .whitesource or .mend files now explicitly lists available options, including those that previously relied only on defaults, making it easier for users to understand and adjust the settings.
    Note: enableRemediation is intentionally not part of the onboarding template, as it requires activating the AI Usage toggle in the Mend AI Native AppSec Platform.

  • Reading HTTP Handler Methods from an array is now supported for JavaScript/TypeScript and the corresponding methods are treated as entry points into the code.

  • Removed the obsolete Lines of Code column from the SAST results table in the Mend AI Native AppSec Platform to streamline the interface and improve clarity of the displayed information.

Resolved Issues

  • Fixed an issue where users with the Security Analyst role could not save scan configuration changes.

Mend AI: Version 25.10.1 (19-October-2025)

2025-10-19T00:00:00+00:00

New Features and Updates

  • (Premium) Developers and DevOps teams can now trigger red-teaming to receive immediate feedback on AI security issues, directly from the CLI or CI/CD jobs using a new API endpoint: /api/v3.0/projects/{projectUuid}/ai/redteam/trigger
    This frictionless integration accelerates remediation and ensures every release meets security standards.

  • (Premium) Introducing two new API endpoints for listing AI components and findings:

    • GET /api/v3.0/projects/{projectUuid}/ai/model/findings/security

    • GET /api/v3.0/applications/{applicationUuid}/ai/model/findings/security
