Mend.io Unified Release Notes

Mend AppSec Platform: Version 25.10.1 (20-October-2025)

2025-10-20T00:00:00+00:00

New Features and Updates

  • The Mend Security Assistant MCP is now available for both SAST and SCA!
    The assistant is an innovative tool integrated into AI-native IDEs, enabling developers to detect and fix vulnerable libraries or pieces of code in real-time.
    This offering allows developers to maintain secure coding practices seamlessly within their existing workflow, securing dependencies and code without interruption and building greater trust in generated outputs. Mend Security Assistant MCP is available for:

    • Amazon Q

    • Claude Code

    • Cursor IDE

    • VS Code Copilot

    • Windsurf IDE

  • Added a new Scan Metadata operational triggering event, enabling users to leverage tags as workflow conditions. You can now define workflows using Scan Tag (Key) or Scan Tag (Key:Value) with Matches and Doesn't Match operatorsideal for assigning scope, labeling branches, or aligning policies by tag.

  • Introducing Persona-Based Violation Filtering: Security and Legal teams can now view violations separately, ensuring each persona sees only relevant risks. Toggle between views or combine them for a unified perspective.

  • Users can now suppress violating findings directly from the main table and side-panel - at the Origin level (Root Library for SCA, Package for Containers), Library level, and Finding level - for improved remediation tracking across SCA, Containers, and Code.

  • Added an Automatic User Creation toggle in the SAML settings. Admins can now choose whether new users are created automatically on first SSO login or must be added manually.

Mend SAST: Version 25.9.1 (29-September-2025)

2025-09-29T00:00:00+00:00

New Features and Updates

  • (Open Beta) Mend SAST now supports Rust. Click here to learn about the Rust coverage.

  • All code snippets are encrypted by default, to strengthen security and prevent unauthorized access. However, now customers may provide their own encryption key via a new API endpoint.

Resolved Issues

  • Security improvements: Multiple core libraries have been updated to their latest secure versions.

  • Fixed an issue where the analysis depth was sometimes overly restricted. This could lead to inconsistencies between incremental and full scans, especially in Fast Scan mode. With this fix, scan results are now more consistent across modes, and in some cases additional findings may be reported.

Mend Developer Platform: Version 25.10.1 (19-October-2025)

2025-10-19T00:00:00+00:00

New Features and Updates

  • The Mend Developer Platform for GitHub is now generally available.

  • Users with the Azure DevOps organization administrator status can now add/remove delegated organization administrators via the Developer Platform UI.
    Delegated organization administrators have the same level of control in the Developer Platform as Organization Administrators, minus the ability to delegate.

  • A new Scan submodules toggle in the Dependencies configuration page allows users to configure submodules to be included/excluded in SCA scans.
    This toggle is not currently available for Azure DevOps.

  • (SAST) When creating the first pull request that includes findings in a repository with no prior scans on the base branch, the security check will now fail, and those findings will not be considered backlog findings. This change enables a smoother onboarding process by allowing new repositories to be initialized through the first pull request instead of requiring a direct commit to the main branch.

Mend for GitHub.com: Version 25.10.1 (19-October-2025)

2025-10-19T00:00:00+00:00

Unified Agent 25.7.2-196 | Renovate 41.97.10 | Remediate 25.10.1 | Pre-Scan Builder (PSB) 25.9.1

New Features and Updates

  • (SAST) Introducing a new approval process for security finding suppressions in code repositories. Developers can propose suppressions, while security managers can approve or reject them, ensuring control without development delays. This bridges developer and security workflows, enhancing efficiency.

  • (SAST) The onboarding configuration template for SCMs using .whitesource or .mend files now explicitly lists available options, including those that previously relied only on defaults, making it easier for users to understand and adjust the settings.
    Note: enableRemediation is intentionally not part of the onboarding template, as it requires activating the AI Usage toggle in the Mend AI Native AppSec Platform.
