Mend.io Unified Release Notes

Mend AppSec Platform: Version 26.4.2 (03-May-2026)

2026-05-03T00:00:00+00:00

New Features and Updates

  • UI Improvements:

    • Updated the administration screens to feature a modern floating bulk actions toolbar at the bottom, making bulk actions more visible and accessible. This change streamlines the user experience by consolidating selected item counts and available actions into a single, easy-to-reach toolbar, ensuring consistency and reducing clicks across all administration screens.

Resolved Issues

  • Fixed an issue where violations linked to a deleted workflow were not removed, which caused data inconsistencies. Now, when a workflow is deleted, all associated violations are properly deleted to maintain data integrity.

  • Fixed an issue where manual Jira issue creation from Mend for Jira Cloud displayed a success message but failed to create a Jira ticket if the required issue type was missing or misconfigured in the target Jira project.

  • Fixed an issue where workflow email notifications weren't sent, ensuring that notifications are now delivered as expected when workflow conditions are met.

  • Fixed an issue where filtering the File Path column in the Source Files tab of the OS Inventory did not return results as expected. Filtering now works correctly, allowing users to search for files by path.

  • Fixed an issue where the Mend Advise browser extension would display a timeout error when analyzing the NJsonSchema 11.0.2 package.

Mend SCA: Version 26.3.2 (12-April-2026)

2026-04-12T00:00:00+00:00

New Features and Updates

  • Introducing new Zero-Day response workflows in the Mend AppSec Platform, offering immediate risk visibility, audit-level transparency, and user-controlled enforcement for critical vulnerabilities. Users benefit from prominent alerts, a dedicated Zero-Day catalog, configurable workflows, and permanent violation tracking—streamlining investigation and remediation without added operational burden.

  • Improved license accuracy of Maven libraries. Updates include clearer license version specificity, removal of redundant entries, and detection of previously unknown licenses.

  • The Due Diligence report can now be generated in Excel format.

Resolved Issues

  • Fixed an issue where certain "software.amazon.awssdk" components displayed duplicated or incorrect copyright statements. All affected components now show accurate and consistent copyright information.

Mend SAST: Version 26.4.2 (04-May-2026)

2026-05-04T00:00:00+00:00

New Features and Updates

  • Added taint analysis support for Python smbclient to enhance detection of path traversal vulnerabilities.

  • Added support to ensure that SSRF vulnerabilities in JavaScript are reported only when detected on the server side, improving the accuracy of security findings.

Mend Container: Version 26.4.2 (03-May-2026)

2026-05-03T00:00:00+00:00

Resolved Issues

  • Fixed an issue where the bson package version was incorrectly detected as UNKNOWN, resulting in a false positive for CVE-2020-7610. The scanner now accurately identifies bson@1.1.4 and avoids reporting this vulnerability when the correct version is present.

Mend AI: Version 25.12.1 (05-January-2026)

2026-01-05T00:00:00+00:00

New Features and Updates

  • (Premium / Core) Introducing System Prompt Risk (open beta), a new detection and remediation offering by Mend AI, for mitigating risks posed by system prompts used in conversational AI interfaces.

    • A new System Prompt Risk table inventories system prompts and provides quick export/sharing and deep-link side panel for prompt context.

    • System Prompt Risks are integrated into the existing AI Security Risk Factors across Projects and Applications. The classification appears as a Conversational Interface chip, is filterable, and automatically participates in dashboards and workflows.

    • Remediation in the form of a hardened system prompt is available in the System Prompt side panel, providing clear, copy-ready remediation guidance for AppSec engineers and developers.

    • The AI Security Dashboard has been enriched with system prompt risk data.

    • System prompt risk data is also available via API.

Mend CLI: Version 26.4.2 (04-May-2026)

2026-05-04T00:00:00+00:00

Resolved Issues

  • Fixed an issue where the CLI logged missing Python packages from requirements.txt at the DEBUG level instead of WARNING.

  • Fixed an issue where the CLI would incorrectly mark scans of invalid projects as failed. Now, such projects are only marked as failed in strict mode; in non-strict mode, a warning is logged without failing the scan.

Mend Unified Agent: Version 26.3.2 (14-April-2026)

2026-04-14T00:00:00+00:00

Resolved Issues

  • Fixed an issue where the Unified Agent failed to detect npm dependencies in Yarn Berry (v4) projects using workspaces. The scanner now correctly identifies and processes workspace package.json files and root yarn.lock files, ensuring all dependencies are accurately reported in scan results.

  • Fixed an issue where configuring python.pipPath to an absolute path caused setup.py dependency resolution to detect only one dependency instead of the full dependency tree. All dependencies are now correctly identified during scans when using an absolute pip path.

Mend Developer Platform: Version 26.4.2 (03-May-2026)

2026-05-03T00:00:00+00:00

Resolved Issues

  • Fixed an issue where the installed repositories table in the Developer Platform was not updating to reflect changes from GitHub, causing UI inconsistencies in certain scenarios. The sync process was improved to ensure accurate repository data and consistent UI experience.

Mend for GitHub.com: Version 26.4.2 (03-May-2026)

2026-05-03T00:00:00+00:00

New Features and Updates

  • Suppressions and suppression requests can now be triggered via a special comment rather than a checkbox, giving developers the ability to include additional context with each suppression.

    • The configuration file now supports a new issueType setting — findingIncludingPullRequests — which, when enabled, creates GitHub issues for all detected findings, including those from pull requests to base branches.

    • This setting also unlocks optional two-way communication: Comments posted in the Mend AppSec Platform appear on the corresponding GitHub issue, and comments made on the GitHub issue that contain a special command are reflected in the Mend AppSec Platform. This keeps developers and security teams in their respective environments while remaining in sync.

Mend for GitHub Enterprise: Version 26.4.1.1 (19-April-2026)

2026-04-19T00:00:00+00:00

Unified Agent 26.3.2 | Renovate 43.102.11 | Remediate 26.4.1 | Pre-Scan Builder (PSB) 25.8.1

New Features and Updates

  • Added support for disabling specific package manager resolvers using the {PACAKGE_MANAGER}.resolveDependencies parameter, allowing more granular control over dependency resolution.

  • Introduced a lower priority queue for Reachability "zero day" scans. This enhancement prevents system overload and ensures more reliable scan processing for enterprise customers with large deployments during periods of high CVE alert activity.

Resolved Issues

  • Fixed an issue where the clone, fetch, and checkout commands in the scanner were limited to a fixed 15‑minute timeout, which caused failures with large repositories. Timeouts are now configurable through the environment variables listed below, providing better support for large repositories.

    • GIT_CLONE_TIMEOUT_MINS 

    • GIT_FETCH_TIMEOUT_MINS 

    • GIT_CHECKOUT_TIMEOUT_MINS
