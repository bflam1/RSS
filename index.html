Mend.io Unified Release Notes

Mend AppSec Platform: Version 25.8.2 (07-September-2025)

2025-09-07T00:00:00+00:00

New Features and Updates

  • Added the ability to define custom SLA values for violations in automation workflows, ranging from 1 - 1000 days.

  • The Mend AI Native AppSec platform now supports IP allowlisting, configurable via the Administration > General tab. Administrators can configure access by specifying exact IP addresses or CIDR ranges, blocking unauthorized IPs from accessing the API or the platform UI.

Mend SCA: Version 25.9.1 (28-September-2025)

2025-09-28T00:00:00+00:00

New Features and Updates

  • Introducing significant SBOM enhancements:

    • Users can now import Syft, Dependency Track and self-generated SBOMs.

    • The CycloneDX 1.6 format is now supported, both import and export.

    • Improved error handling for SBOM imports.

    • Improved Maven matches when importing an SBOM.

  • Improved reliability and accuracy of npm package resolution.

Mend SAST: Version 25.9.1 (29-September-2025)

2025-09-29T00:00:00+00:00

New Features and Updates

  • (Open Beta) Mend SAST now supports Rust. Click here to learn about the Rust coverage.

  • All code snippets are encrypted by default, to strengthen security and prevent unauthorized access. However, now customers may provide their own encryption key via a new API endpoint.

Resolved Issues

  • Security improvements: Multiple core libraries have been updated to their latest secure versions.

  • Fixed an issue where the analysis depth was sometimes overly restricted. This could lead to inconsistencies between incremental and full scans, especially in Fast Scan mode. With this fix, scan results are now more consistent across modes, and in some cases additional findings may be reported.

Mend for GitHub Enterprise: Version 25.9.1.2 (08-October-2025)

2025-10-08T00:00:00+00:00

Unified Agent 25.7.2-196 | Renovate 41.97.10 | Remediate 25.9.1 | Pre-Scan Builder (PSB) 25.8.1

New Features and Updates

  • If LVP (Least Vulnerable Package) remediation strategy is enabled, the security check will show the suggested fix accordingly.

  • Introducing a new API endpoint for retrieving the number of pending webhooks in the system. This addition provides visibility into the current webhook state and enables users to scale up/down based on webhook backlog.

  • Security check run table improvements: Easier to see if the vulnerability and suggested fix are reported on a direct dependency or a transitive one.

Resolved Issues

  • Fixed an issue where the scanner didnt respect proxy configuration in certain setups, leading to scan failure.

  • Upgraded PHP in the scanner to version 8.4.8, which is not susceptible to CVE-2024-4577 as some older PHP versions.

  • Fixed an issue where the scanner was creating root-owned files, leading to temporary directories not being deleted post-use, for improved housekeeping

  • Fixed a bug where using the environment variable MEND_GLOBAL_CUSTOMIZATION_REPO in the controller resulted in the error Failed to get private key from key vault.

  • (SAST) Fixed an issue where the noDefaultExclusions setting in the .mendsastcli-config.json configuration file, used for ignoring predefined exclusions, was ineffective, leading to predefined exclusions not being ignored.
